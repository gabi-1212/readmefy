# Support

## Getting help

- Open a GitHub Discussion for usage questions or adoption advice.
- Open a GitHub Issue when you can point to a reproducible detection or generation bug.
- Use [SECURITY.md](./SECURITY.md) for security-sensitive reports.

## What to include

- repository type and package manager
- relevant checked-in files
- expected README behavior
- actual output or diff summary
- `readmefy doctor --json` output when possible

## Maintainer expectations

`readmefy` is infrastructure-focused. Reports that include the minimal repository signals needed to reproduce a problem will move fastest.
