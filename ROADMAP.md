# Roadmap

`readmefy` is focused on becoming trustworthy repository documentation infrastructure.

## Near term

- Add richer workspace rollups for monorepos
- Improve diff summaries for CI failures
- Expand Python, Rust, and Go metadata extraction
- Add more marker-aware merge scenarios
- Tighten config diagnostics in `doctor`

## Adoption features

- Template presets for docs-heavy OSS libraries
- Optional badge provider customization
- Richer environment variable descriptions from comments
- Configurable project structure relevance rules
- Conventional commit release notes integration

## Good first issues

- Add support for `docker-compose.yaml`
- Detect Bun lockfiles in fixtures and docs
- Improve README license wording when only a package manifest is present
- Add tests for mixed `README.md` line endings
- Add more App Router-specific examples for Next.js fixtures
