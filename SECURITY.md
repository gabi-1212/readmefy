# Security Policy

## Supported versions

| Version | Supported |
| ------- | --------- |
| 0.x     | Yes       |

## Reporting a vulnerability

Please use GitHub's private vulnerability reporting on the repository Security tab. Do not open a public issue for a security report.

Include:

- impact and affected command paths
- reproduction steps
- whether the issue requires a malicious repository or malicious config
- any proposed mitigation

You can expect an acknowledgement within three business days and status updates as the investigation progresses.
